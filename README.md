1. Библиотека - набор функций которые можно импортировать в проект и использовать
2. Фреймворк - говорит о том как в целом должна выглядеть структура файлов.
Ее нужно соблюдать (например все файлы, которые содержат тесты, должны начинаться с префикса "test_"

"def" - определяет начало теста
() - оператор функции
Тестовые фреймворки обычно состоят из трех столпов:
1. Collector (ищет тесты по каким-либо признакам)
2. Runner (движок тестового фреймворка, который запускает тесты. Может делать это по разному)
3. Reporter (показывает результаты выполнения тестов)

ФИКСТУРЫ
Хронятся в отдельном файле conftest.py
Зависимости между фикстурами обозначаются в скобках самих фикстур
Для конкретного теста необходимые фикстуры обозначаются в () через запятую

Например:
@pytest.fixture
def page(browser):
Таким образом мы задаем порядок, и перед выполнением фикстуры page обязательно должна выполниться фикстура browser

@pytest.fixture - обозначение для фикстур (предусловий)

yield - обозначение для фикстуры, которые должны выполняться после прохождение теста.
Все что написано после этого слова будет использовано как постусловие
